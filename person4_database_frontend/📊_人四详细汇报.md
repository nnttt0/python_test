## 2. 数据库持久化实现

### 2.1 数据库连接管理（db.py）

#### 核心概念解释

**什么是连接池？**
- 📖 **类比**：就像餐厅准备了10个水杯，客人来了直接拿，不用每次现洗杯子
- 💡 **作用**：避免频繁创建和销毁数据库连接，提高性能
- 🔧 **配置**：`pool_size=10`表示常备10个连接，`max_overflow=20`表示最多可额外创建20个

**什么是Session？**
- 📖 **类比**：就像你和数据库之间的"对话窗口"
- 💡 **作用**：通过Session执行所有数据库操作（查询、插入、更新、删除）
- ⚠️ **注意**：用完必须关闭（`db.close()`），否则会泄露连接

#### 代码详解

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

# 第1步：创建数据库引擎（连接池）
engine = create_engine(
    "mysql+pymysql://root:password@localhost:3306/stroke_prediction",
    pool_size=10,           # 连接池大小：常备10个连接
    max_overflow=20,        # 最大溢出：最多再创建20个
    pool_recycle=3600       # 连接回收时间：1小时未使用则断开重连
)

# 第2步：创建Session工厂
SessionLocal = sessionmaker(
    autocommit=False,       # 不自动提交（需要手动commit）
    autoflush=False,        # 不自动刷新
    bind=engine             # 绑定到上面创建的引擎
)

# 第3步：创建ORM基类
Base = declarative_base()   # 所有模型类都要继承这个基类

# 第4步：依赖注入函数（供Flask使用）
def get_db():
    """获取数据库Session"""
    db = SessionLocal()
    try:
        yield db            # 生成器：每次调用返回新的Session
    finally:
        db.close()          # 确保Session被关闭

# 第5步：初始化数据库
def init_db():
    """创建所有表结构"""
    Base.metadata.create_all(bind=engine)
```

#### 关键知识点

| 概念 | 说明 | 示例 |
|------|------|------|
| `create_engine` | 创建数据库引擎（连接池） | `engine = create_engine("mysql+pymysql://...")` |
| `sessionmaker` | 创建Session工厂 | `SessionLocal = sessionmaker(bind=engine)` |
| `declarative_base` | ORM基类 | `Base = declarative_base()` |
| `yield` | 生成器关键字 | 用于依赖注入，确保资源清理 |
| `pool_size` | 连接池大小 | 根据并发量调整（默认10） |

### 2.2 ORM模型定义（models.py）

#### 核心概念解释

**什么是ORM？**
- 📖 **全称**：Object-Relational Mapping（对象关系映射）
- 💡 **作用**：用Python类操作数据库表，不用写SQL语句
- 🎯 **好处**：
  - ✅ 防止SQL注入
  - ✅ 代码更易读
  - ✅ 支持多种数据库（MySQL、PostgreSQL、SQLite）

**对比示例**：

```python
# ❌ 传统SQL方式（容易出错）
cursor.execute("INSERT INTO predictions (gender, age) VALUES ('%s', %d)" % (gender, age))

# ✅ ORM方式（安全、清晰）
record = PredictionRecord(gender='Male', age=45)
db.add(record)
db.commit()
```

#### 代码详解

```python
from sqlalchemy import Column, Integer, String, Float, DateTime, Text, JSON
from sqlalchemy.sql import func
from database.db import Base  # 继承ORM基类

class PredictionRecord(Base):
    """预测记录表 - 对应数据库中的predictions表"""
    
    __tablename__ = 'predictions'  # 表名
    
    # ===== 主键字段 =====
    id = Column(Integer, primary_key=True, autoincrement=True)
    #               ↑ 数据类型    ↑ 主键         ↑ 自增
    
    # ===== 患者基本信息（10个特征字段）=====
    gender = Column(String(20), nullable=False)      # 性别：Male/Female
    age = Column(Float, nullable=False)              # 年龄：0-100
    hypertension = Column(Integer, default=0)        # 高血压：0=无，1=有
    heart_disease = Column(Integer, default=0)       # 心脏病：0=无，1=有
    ever_married = Column(String(10))                # 婚姻状况：Yes/No
    work_type = Column(String(50))                   # 工作类型：Private/Govt/Self-employed等
    residence_type = Column(String(20))              # 居住类型：Urban/Rural
    avg_glucose_level = Column(Float)                # 平均血糖水平
    bmi = Column(Float)                              # BMI指数
    smoking_status = Column(String(50))              # 吸烟状况：smokes/never smoked/formerly smoked
    
    # ===== 预测结果字段 =====
    risk_score = Column(Float, nullable=False)       # 风险分数：0-1之间的小数
    risk_percent = Column(Float, nullable=False)     # 风险百分比：0-100
    risk_level = Column(String(20), nullable=False)  # 风险等级：safe/warning/danger
    suggestion = Column(Text)                        # 建议文本（长文本用Text类型）
    
    # ===== 元数据字段 =====
    prediction_type = Column(String(20), default='single')  # 预测类型：single/batch
    batch_id = Column(String(50))                    # 批次ID（批量预测时使用）
    created_at = Column(DateTime, server_default=func.now())  # 创建时间（自动填充当前时间）
    extra_info = Column(JSON)                        # 额外信息（JSON格式，灵活存储）
    
    def to_dict(self):
        """将记录转换为字典（方便转JSON返回给前端）"""
        return {
            'id': self.id,
            'gender': self.gender,
            'age': self.age,
            # ... 其他字段
            'created_at': self.created_at.isoformat() if self.created_at else None
        }
```

#### 字段类型对照表

| SQLAlchemy类型 | MySQL类型 | 适用场景 | 示例 |
|---------------|----------|---------|------|
| `Integer` | INT | 整数ID、计数 | `id`, `hypertension` |
| `Float` | FLOAT | 小数数值 | `age`, `bmi`, `risk_score` |
| `String(20)` | VARCHAR(20) | 短文本 | `gender`, `risk_level` |
| `Text` | TEXT | 长文本 | `suggestion` |
| `DateTime` | DATETIME | 日期时间 | `created_at` |
| `JSON` | JSON | 结构化数据 | `extra_info` |

#### 约束条件

| 参数 | 作用 | 示例 |
|------|------|------|
| `primary_key=True` | 设为主键 | `id`字段 |
| `autoincrement=True` | 自动递增 | 每次插入自动生成新ID |
| `nullable=False` | 不允许为空 | 必填字段 |
| `default=0` | 默认值 | 未填写时自动设为0 |
| `server_default=func.now()` | 服务器默认值 | 自动填充当前时间 |

### 2.3 CRUD服务封装（services.py）

#### 核心概念解释

**什么是CRUD？**
- 📖 **全称**：Create（创建）、Read（读取）、Update（更新）、Delete（删除）
- 💡 **作用**：封装所有数据库操作，提供简洁的API
- 🎯 **好处**：业务逻辑层不需要关心SQL细节

#### 代码详解

##### （1）保存单次预测记录

```python
@staticmethod
def save_single_prediction(db: Session, input_data: Dict, result: Dict) -> PredictionRecord:
    """
    保存单次预测记录
    
    参数：
    - db: 数据库Session（由app.py传入）
    - input_data: 用户输入的表单数据（字典）
    - result: 模型预测结果（字典）
    
    返回：
    - 保存后的PredictionRecord对象
    """
    # 第1步：创建记录对象
    record = PredictionRecord(
        gender=input_data.get('gender', ''),          # .get()方法：如果键不存在返回默认值''
        age=float(input_data.get('age', 0)),          # float()转换：确保是浮点数
        hypertension=int(input_data.get('hypertension', 0)),  # int()转换：确保是整数
        heart_disease=int(input_data.get('heart_disease', 0)),
        ever_married=input_data.get('ever_married', ''),
        work_type=input_data.get('work_type', ''),
        residence_type=input_data.get('residence_type', ''),
        avg_glucose_level=float(input_data.get('avg_glucose_level', 0)) if input_data.get('avg_glucose_level') else None,
        #                                                                                              ↑ 处理空值
        bmi=float(input_data.get('bmi', 0)) if input_data.get('bmi') else None,
        smoking_status=input_data.get('smoking_status', ''),
        
        # 预测结果字段
        risk_score=result.get('risk', 0),
        risk_percent=result.get('risk_percent', 0),
        risk_level=result.get('level', 'safe'),
        suggestion=result.get('suggestion', ''),
        
        prediction_type='single'  # 标记为单次预测
    )
    
    # 第2步：添加到Session
    db.add(record)
    
    # 第3步：提交事务（真正写入数据库）
    db.commit()
    
    # 第4步：刷新对象（获取自动生成的id和created_at）
    db.refresh(record)
    
    # 第5步：返回保存的记录
    return record
```

**关键点解析**：

1. **为什么用`.get()`而不是`[]`？**
   ```python
   # ❌ 如果键不存在会报错
   gender = input_data['gender']  # KeyError!
   
   # ✅ 如果键不存在返回默认值
   gender = input_data.get('gender', '')  # 安全
   ```

2. **为什么要类型转换？**
   ```python
   # 前端传来的数据都是字符串
   input_data = {'age': '45', 'hypertension': '1'}
   
   # 数据库需要正确的类型
   age = float('45')      # 45.0
   hypertension = int('1')  # 1
   ```

3. **事务提交的含义**
   ```python
   db.add(record)    # 只是添加到Session（内存中）
   db.commit()       # 真正执行SQL INSERT语句（写入数据库）
   ```

##### （2）保存批量预测记录

```python
@staticmethod
def save_batch_predictions(db: Session, records: List[Dict], batch_id: str = None) -> List[PredictionRecord]:
    """
    保存批量预测记录
    
    参数：
    - records: 多条记录数据的列表
    - batch_id: 批次ID（用于关联同一批次的记录）
    """
    # 第1步：生成批次ID（如果没有提供）
    if not batch_id:
        batch_id = str(uuid.uuid4())[:8]  # 例如：'a3f2b1c4'
    
    saved_records = []
    
    # 第2步：循环创建记录
    for record_data in records:
        record = PredictionRecord(
            # ... 字段赋值同上
            prediction_type='batch',  # 标记为批量预测
            batch_id=batch_id         # 关联批次ID
        )
        db.add(record)
        saved_records.append(record)
    
    # 第3步：一次性提交所有记录（事务）
    db.commit()
    
    # 第4步：刷新所有对象
    for record in saved_records:
        db.refresh(record)
    
    return saved_records
```

**关键点解析**：

1. **批次ID的作用**
   - 📖 **类比**：就像快递单号，可以追踪同一批的所有包裹
   - 💡 **用途**：查询某次批量预测的所有记录
   ```python
   # 查询某个批次的所有记录
   batch_records = db.query(PredictionRecord).filter(PredictionRecord.batch_id == 'a3f2b1c4').all()
   ```

2. **事务的重要性**
   ```python
   # ✅ 要么全部成功，要么全部失败（原子性）
   for record_data in records:
       db.add(record)
   db.commit()  # 一次性提交
   
   # ❌ 如果中间出错，会导致部分数据写入
   for record_data in records:
       db.commit()  # 每条单独提交（不推荐）
   ```

##### （3）查询操作

```python
@staticmethod
def get_prediction_by_id(db: Session, prediction_id: int) -> Optional[PredictionRecord]:
    """根据ID获取单条记录"""
    return db.query(PredictionRecord).filter(PredictionRecord.id == prediction_id).first()
    #            ↑ 查询对象              ↑ 过滤条件                     ↑ 取第一条

@staticmethod
def get_predictions_by_batch(db: Session, batch_id: str) -> List[PredictionRecord]:
    """根据批次ID获取所有记录"""
    return db.query(PredictionRecord).filter(PredictionRecord.batch_id == batch_id).all()
    #                                                                        ↑ 取所有

@staticmethod
def get_all_predictions(db: Session, limit: int = 100, offset: int = 0) -> List[PredictionRecord]:
    """获取所有记录（分页）"""
    return db.query(PredictionRecord)\
             .order_by(PredictionRecord.created_at.desc())\  # 按时间倒序
             .limit(limit)\                                  # 限制数量
             .offset(offset)\                                # 偏移量（分页）
             .all()

@staticmethod
def get_statistics(db: Session) -> Dict:
    """获取预测统计数据"""
    total = db.query(PredictionRecord).count()
    high_risk = db.query(PredictionRecord).filter(PredictionRecord.risk_level == 'danger').count()
    medium_risk = db.query(PredictionRecord).filter(PredictionRecord.risk_level == 'warning').count()
    low_risk = db.query(PredictionRecord).filter(PredictionRecord.risk_level == 'safe').count()
    
    return {
        'total': total,
        'high_risk': high_risk,
        'medium_risk': medium_risk,
        'low_risk': low_risk,
        'high_risk_rate': round(high_risk / total * 100, 2) if total > 0 else 0
    }
```

### 3.1 JavaScript基础语法速成

#### async/await（异步编程）

**什么是异步？**
- 📖 **类比**：就像你去餐厅点餐，不用一直站在柜台等，可以先找座位坐下，餐好了服务员会叫你
- 💡 **作用**：避免页面卡死，让用户可以继续操作

```javascript
// ❌ 同步方式（会卡死页面）
const response = fetch('/predict');  // 等待2秒...
const data = response.json();        // 页面完全卡住，用户无法点击任何按钮

// ✅ 异步方式（不会卡死）
async function predict() {
    const response = await fetch('/predict');  // 等待期间页面仍可响应
    const data = await response.json();        // 数据到了再继续执行
    console.log(data);
}
predict();  // 立即返回，不阻塞页面
```

**关键点**：
1. `async`关键字：声明函数是异步的
2. `await`关键字：等待Promise完成（只能在async函数中使用）
3. `fetch()`返回Promise对象

### 3.2 单条预测功能（main.js 68-101行）

#### 完整代码+逐行注释

```javascript
// 第1步：找到表单元素，绑定submit事件
document.getElementById('predictForm').addEventListener('submit', async (e) => {
    e.preventDefault();  // 阻止表单默认提交行为（防止页面刷新）
    
    showLoading(true);  // 显示加载动画
    
    // 第2步：收集表单数据
    const formData = new FormData(e.target);  // e.target就是表单本身
    
    try {
        // 第3步：发送HTTP请求到后端
        const response = await fetch('/predict', {
            method: 'POST',      // HTTP方法：POST
            body: formData       // 请求体：表单数据
        });
        
        // 第4步：解析JSON响应
        const data = await response.json();
        
        // 第5步：判断是否成功
        if (data.success) {
            // 第6步：动态生成结果HTML
            document.getElementById('predictResult').innerHTML = `
                <div class="risk-card ${data.level}">
                    <h3>评估结果</h3>
                    <div class="risk-percent">${data.risk_percent}%</div>
                    <div class="risk-bar-container">
                        <div class="risk-bar" style="width: ${data.risk_percent}%"></div>
                    </div>
                    <p style="font-weight:500;margin-top:8px;">${data.suggestion}</p>
                    <p style="font-size:13px;margin-top:8px;opacity:0.9;">${data.message}</p>
                </div>
            `;
            //          ↑ 模板字符串：${data.level}会被替换为'danger'或'warning'或'safe'
        } else {
            // 第7步：显示错误信息
            alert('预测失败：' + data.error);
        }
    } catch (error) {
        // 第8步：捕获网络错误
        alert('请求失败：' + error);
    }
    
    showLoading(false);  // 隐藏加载动画
});
```

function bindChartButtons() {
    // 选择所有图表按钮
    document.querySelectorAll('.chart-btn').forEach(btn => {
        // 移除旧的事件监听器（防止重复绑定）
        btn.removeEventListener('click', btn._listener);
        
        // 获取图表类型
        const chartType = btn.dataset.chart;  // 从data-chart属性读取
        
        // 创建新的事件监听器
        const listener = () => generateChart(chartType);
        
        // 绑定事件
        btn.addEventListener('click', listener);
        
        // 保存监听器引用（用于后续移除）
        btn._listener = listener;
    });
}

### 3.6 CSV导出功能（main.js 190-217行）

#### 核心代码

```javascript
document.getElementById('exportBtn').addEventListener('click', async () => {
    if (!currentData) return;  // 没有数据则退出
    
    showLoading(true);
    
    try {
        // 第1步：发送导出数据请求
        const response = await fetch('/export', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify(currentData)  // 发送之前保存的数据
        });
        
        const data = await response.json();
        
        if (data.success) {
            // 第2步：创建Blob对象（二进制大对象）
            const blob = new Blob([data.csv_data], {type: 'text/csv;charset=utf-8;'});
            //                                    ↑ 指定MIME类型和编码
            
            // 第3步：创建下载链接
            const link = document.createElement('a');
            link.href = URL.createObjectURL(blob);
            //                    ↑ 创建临时URL指向Blob
            link.download = '中风风险预测结果.csv';
            
            // 第4步：触发下载
            link.click();
            
            // 第5步：释放URL对象（清理内存）
            URL.revokeObjectURL(link.href);
        } else {
            alert('导出失败：' + data.error);
        }
    } catch (error) {
        alert('导出失败：' + error);
    }
    
    showLoading(false);
});
```

**关键点**：
- `Blob`：Binary Large Object，用于存储二进制数据
- `URL.createObjectURL()`：创建临时URL，指向内存中的Blob
- `URL.revokeObjectURL()`：释放URL，避免内存泄漏

## 6. 常见问题解答

### Q1: 为什么数据库连接要关闭？

**A**: 数据库连接是有限资源，不关闭会导致连接池耗尽。

```python
# ❌ 错误做法：忘记关闭
db = SessionLocal()
db.add(record)
db.commit()
# 忘记 db.close() → 连接泄露

# ✅ 正确做法：使用try-finally
db = SessionLocal()
try:
    db.add(record)
    db.commit()
finally:
    db.close()  # 确保关闭
```

**类比**：就像借书不还，图书馆的书会被借光。

---

### Q2: async/await和普通函数有什么区别？

**A**: async函数不会阻塞页面，用户可以继续操作。

```javascript
// ❌ 同步方式（会卡死页面）
function predict() {
    const response = fetch('/predict');  // 等待2秒...
    const data = response.json();        // 页面完全卡住
    console.log(data);
}
predict();
console.log('这行要等2秒后才执行');

// ✅ 异步方式（不会卡死）
async function predict() {
    const response = await fetch('/predict');  // 等待期间页面仍可响应
    const data = await response.json();
    console.log(data);
}
predict();
console.log('这行立即执行，不等fetch完成');
```

---

### Q3: 为什么要用ORM而不是直接写SQL？

**A**: ORM更安全、更易维护。

```python
# ❌ 直接写SQL（容易SQL注入）
cursor.execute(f"INSERT INTO predictions (age) VALUES ({age})")
# 如果 age = "0; DROP TABLE predictions;" → 数据库被删除！

# ✅ ORM（防止SQL注入）
record = PredictionRecord(age=age)
db.add(record)
db.commit()
# SQLAlchemy会自动转义特殊字符
```

### Q4: Base64编码的图片为什么能直接在HTML中显示？

**A**: 浏览器支持Data URL格式。

```html
<!-- 传统方式：图片存储在服务器 -->
<img src="/static/images/chart.png">

<!-- Base64方式：图片嵌入HTML -->
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...">
<!--     ↑ 协议      ↑ MIME类型    ↑ Base64数据 -->
```

**优点**：
- ✅ 不需要额外的HTTP请求
- ✅ 图片随HTML一起传输
- ✅ 适合小图片（如图表）

**缺点**：
- ❌ 体积增大33%
- ❌ 不适合大图片

async function predict() {
